# W-63: DNS 서비스 동적 업데이트 설정 점검

## 1. 개요

이 디렉터리는 KISA의 주요정보통신기반시설 기술적 취약점 가이드라인의 `W-63: DNS 서비스 구동 점검` 항목에 대한 자동화 스크립트와 관련 문서를 포함합니다.

DNS 동적 업데이트 기능이 안전하지 않게 ('없음'이 아닌 '안전' 또는 '안전하지 않음') 설정될 경우, 공격자가 DNS 레코드를 임의로 변조하여 사용자를 악성 사이트로 유도하거나(DNS Poisoning) 내부 시스템 통신을 가로챌 수 있는 심각한 위협이 존재합니다.

## 2. 파일 설명

- `W-63.ps1`
  - **설명**: 메인 점검 및 조치 스크립트입니다. 시스템의 주 DNS 영역(Primary Zone)의 동적 업데이트 설정을 확인하여 `None`(없음)이 아닌 경우 취약점으로 진단하고, 발견 시 자동으로 조치합니다.
  - **실행 전제 조건**: Windows Server, DNS 서버 역할 설치, 관리자 권한

- `set_vulnerability_W-63.md`
  - **설명**: 테스트 목적으로 시스템을 의도적으로 취약한 상태로 만드는 수동 절차를 안내합니다.

## 3. 사용법

### 3.1. 취약점 점검 및 자동 조치

`W-63.ps1` 스크립트는 `-a` 매개변수를 통해 세 가지 모드로 실행될 수 있습니다: `Detect`, `Remediate`, `Restore`. 기본값은 `Detect`입니다.

1.  PowerShell을 **관리자 권한**으로 실행합니다.
2.  스크립트가 있는 디렉터리로 이동합니다.
    ```powershell
    cd W-63
    ```

3.  **점검 모드 (Detect - 기본값)**
    *   시스템의 DNS 동적 업데이트 설정 취약점만을 점검합니다.
    *   실행 명령:
        ```powershell
        .\W-63.ps1
        ```

4.  **조치 모드 (Remediate)**
    *   취약점을 점검하고, 발견된 취약점에 대해 동적 업데이트 설정을 'None'으로 변경합니다.
    *   실행 명령:
        ```powershell
        .\W-63.ps1 -a Remediate
        ```

5.  **복원 모드 (Restore)**
    *   이전에 `Remediate` 모드에서 백업된 설정을 사용하여 원래 상태로 복원합니다.
    *   실행 명령:
        ```powershell
        .\W-63.ps1 -a Restore
        ```

6.  모든 실행 결과는 `KISA_RESULT/` 디렉터리에 로그 및 JSON 파일로 저장됩니다.

### 3.2. 취약점 상태 재현 (테스트용)

테스트를 위해 시스템을 의도적으로 취약한 상태로 만들려면 `set_vulnerability_W-63.md` 를 읽고 진행하세요.

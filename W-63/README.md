# W-63: DNS 서비스 동적 업데이트 설정 점검

## 1. 개요

이 디렉터리는 KISA의 주요정보통신기반시설 기술적 취약점 가이드라인의 `W-63: DNS 서비스 구동 점검` 항목에 대한 자동화 스크립트와 관련 문서를 포함합니다.

DNS 동적 업데이트 기능이 안전하지 않게 ('없음'이 아닌 '안전' 또는 '안전하지 않음') 설정될 경우, 공격자가 DNS 레코드를 임의로 변조하여 사용자를 악성 사이트로 유도하거나(DNS Poisoning) 내부 시스템 통신을 가로챌 수 있는 심각한 위협이 존재합니다.

## 2. 파일 설명

- `W-63.ps1`
  - **설명**: 메인 점검 및 조치 스크립트입니다. 시스템의 주 DNS 영역(Primary Zone)들의 동적 업데이트 설정을 확인하여 `None`(없음)이 아닌 경우 취약점으로 진단합니다. 취약점 발견 시 사용자 동의 하에 자동으로 '없음'으로 변경하여 조치하고, 결과를 `KISA_RESULT/W-63.json` 파일로 저장합니다.
  - **실행 전제 조건**: Windows Server, DNS 서버 역할 설치, 관리자 권한

- `set_vulnerabilities.ps1`
  - **설명**: 테스트 목적으로 시스템을 의도적으로 취약한 상태로 만드는 스크립트입니다. 첫 번째 주 DNS 영역의 동적 업데이트 설정을 '안전하지 않음(NonsecureAndSecure)'으로 변경합니다.
  - **경고**: **테스트 및 시연 목적으로만 사용해야 합니다.** 운영 환경에서는 절대 사용하지 마십시오.

- `W-63.md`, `Project_Report.md`, `W-63_Cmdlet_Analysis.md`, `W-63_Script_Analysis.md`
  - **설명**: `W-63.ps1` 스크립트에 대한 상세 설명, 프로젝트 보고서, 전문가용 심층 분석 등 다양한 관점의 문서들입니다. 탐지/수정 원리, PowerShell Cmdlet 분석, MITRE ATT&CK 매핑 등을 포함합니다.

## 3. 사용법

### 3.1. 취약점 점검 및 자동 조치

1.  PowerShell을 **관리자 권한**으로 실행합니다.
2.  스크립트가 있는 디렉터리로 이동합니다.
    ```powershell
    cd W-63
    ```
3.  메인 스크립트를 실행합니다.
    ```powershell
    .\W-63.ps1
    ```
4.  스크립트가 취약점을 발견하면 자동으로 조치할지 묻습니다. `y`를 입력하면 취약한 모든 DNS 영역의 동적 업데이트 설정이 'None'으로 변경됩니다.
5.  모든 실행 과정은 `KISA_RESULT/W-63.log`에 기록되며, 최종 결과는 `KISA_RESULT/W-63.json` 파일에 저장됩니다.

### 3.2. 취약점 상태 재현 (테스트용)

1.  PowerShell을 **관리자 권한**으로 실행합니다.
2.  스크립트가 있는 디렉터리로 이동합니다.
    ```powershell
    cd W-63
    ```
3.  취약점 재현 스크립트를 실행합니다.
    ```powershell
    .\set_vulnerabilities.ps1
    ```
